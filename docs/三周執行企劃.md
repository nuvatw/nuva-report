# 三週執行企劃：報告視覺一致化與背景設定系統

這份企劃先針對目前 report project 做完整盤點，接著拆成三週可執行計畫。目標不是一次把所有頁面重做，而是用可維護的方式，把「初學 AI MV 課程」與既有三份 report 的視覺語言拉齊，同時保留目前使用者喜歡的格線背景，並把背景控制變成登入後可調整的站內功能。

## 1. 專案現況

目前專案是純靜態網站，主要由首頁、登入頁、報告 HTML、共用 JS 與資料建置腳本組成。

| 區塊 | 目前檔案 | 現況 |
| --- | --- | --- |
| 首頁 | `index.html` | timeline 卡片、搜尋、公開/不公開管理、登入後顯示管理功能。 |
| 登入頁 | `login.html` | 使用 `0812` 登入，狀態存在 `sessionStorage`，關閉頁面後需重新登入。 |
| 報告頁 | `reports/*.html` | 每份 report 都是獨立 HTML，CSS 多數寫在各自頁面內。 |
| 共用報告工具 | `assets/report-reader.js`、`assets/report-actions.js` | 提供左側目錄、私密報告擋板、右下角「更多 AI 課程 / 列印文件」。 |
| 資料生成 | `scripts/build-report-index.mjs` | 從 report metadata 產生 `data/reports.json` 與 `data/reports.js`。 |
| 維運驗證 | `scripts/check-links.mjs`、`scripts/audit-maintenance.mjs`、`scripts/verify-*.mjs` | 已有基本檢查流程，但尚未檢查全站視覺 token 或背景設定。 |

目前四份 report 分別是：

| 報告 | 視覺特徵 | 與本次需求的關係 |
| --- | --- | --- |
| `2026-05-12-nuva-new-taipei.html` | 藍色系、A4 成果書、粗線條、左目錄、清楚頁碼。 | 可作為報告節奏、目錄、頁面密度參考。 |
| `2026-05-22-itri-n8n-lv1.html` | 藍色系、資料型成果報告、表格與分段清楚。 | 可作為理性、教學成果型 report 的結構參考。 |
| `2026-05-27-uni-president-ai-workshop-proposal.html` | 藍綠系、提案感較強、區塊較有層次。 | 可作為商務提案與色彩節制參考。 |
| `2026-05-27-ai-experience-sharing.html` | 米灰格線背景、互動 prompt 區、可勾選任務清單、教學手冊感。 | 本次要保留背景方向，但讓色調與其他 report 更一致。 |

## 2. 問題盤點

### 2.1 視覺一致性

目前三份既有 report 的共同語言是：

- A4 紙張感明確，內容在白色 paper 上閱讀。
- 使用粗黑線、頁碼、頁首資訊建立報告感。
- 主要色彩節制，常見藍色作為可互動與重點色。
- 文字密度偏報告，而不是 landing page。

AI MV report 已經有好用的互動功能，但視覺上目前比較像獨立做出來的一份教學手冊。需要調整的方向是：

- 保留格線背景，但改成更淡的白色與藍色系，維持紙面桌面感。
- 報告內的重點色要與既有三份 report 更接近，減少不同色系互相搶戲。
- 表格、prompt 區、checklist、影片區都要維持實作感，但線條、間距與字級要與既有 report 一致。
- 右下角 action button、左側目錄與登入擋板也要吃同一套背景與色彩設定。

### 2.2 技術維護性

目前最大的維護問題是背景與主色分散在不同 HTML 裡：

- `index.html` 有自己的 body 背景。
- `login.html` 有自己的 body 背景。
- AI MV report 有格線背景。
- 另外三份 report 大多只使用純色 `--desk` 背景。
- `report-reader.js` 裡的私密擋板也直接寫死背景色。

如果直接逐頁改 CSS，短期可以完成，但未來每新增一份 report 又會回到手動調整。因此本次應該把背景抽成「全站共用設定」，再讓每頁套用。

### 2.3 管理功能

首頁目前已有登入後管理公開狀態的功能，技術上適合加上「背景設定」。

目前可沿用：

- 登入狀態：`sessionStorage` 的 `nuva-report-admin-auth`。
- 管理資料儲存：`localStorage`。
- 首頁 admin 區塊：登入後顯示控制項，未登入只顯示「登入」按鈕。

需要新增：

- 背景設定資料結構。
- 所有頁面共用的背景套用腳本。
- 首頁登入後的背景控制 UI。
- 預設值、重設值與防呆驗證。

## 3. 建議架構

### 3.1 新增共用背景腳本

建議新增：

```text
assets/report-theme.js
```

功能：

- 從 `localStorage` 讀取背景設定。
- 若沒有設定，使用預設白藍格線背景。
- 把設定轉成 CSS variables 套到 `document.documentElement`。
- 首頁、登入頁、所有 report 頁都載入同一支腳本。
- 印刷模式自動回到白底，避免 PDF 或列印被背景干擾。

建議儲存 key：

```text
nuva-report-background-settings
```

建議資料格式：

```json
{
  "pattern": "grid",
  "color": "#f8fbff",
  "lineColor": "#1d4ed8",
  "opacity": 0.034,
  "brightness": 1.04,
  "contrast": 0.92,
  "size": 42
}
```

### 3.2 CSS variables

建議所有頁面最後都改成吃這些變數：

```css
:root {
  --report-bg-base: #f8fbff;
  --report-bg-line: rgba(29, 78, 216, .034);
  --report-bg-line-strong: rgba(29, 78, 216, .046);
  --report-bg-size: 42px;
  --report-paper: #fff;
  --report-ink: #172025;
  --report-blue: #1d4ed8;
}
```

背景統一由共用樣式輸出：

```css
body {
  background:
    linear-gradient(90deg, var(--report-bg-line-strong) 1px, transparent 1px),
    linear-gradient(0deg, var(--report-bg-line) 1px, transparent 1px),
    var(--report-bg-base);
  background-size: var(--report-bg-size) var(--report-bg-size);
}
```

### 3.3 背景控制 UI

首頁登入後新增一個簡潔的管理區，不在未登入狀態顯示。

控制項建議：

| 控制項 | UI | 用途 |
| --- | --- | --- |
| 紋路 | segmented control | `格線`、`點陣`、`橫線`、`無紋路`。 |
| 底色 | color input + preset swatches | 調整白色、冷白、淡藍等背景基底。 |
| 紋路透明度 | range slider | 控制線條或點陣明顯程度。 |
| 亮度 | range slider | 讓背景偏亮或偏暗。 |
| 格線大小 | range slider | 控制格線密度。 |
| 重設 | button | 回到預設白藍格線。 |

預設值建議：

- 底色：`#f8fbff`
- 紋路：`grid`
- 透明度：`0.032`
- 亮度：`1.04`
- 格線大小：`42`

這會把 AI MV 背景收斂為白色與藍色，避免再出現橘色或米色偏黃的感覺。

## 4. 三週執行計畫

### 第 1 週：視覺盤點與共用基礎

目標：先把「風格一致」定義清楚，建立共用背景與設計 token，不急著一次改完所有版面。

#### 工作項目

1. 建立 report 視覺盤點文件
   - 比較四份 report 的字體、主色、背景、線條粗細、頁碼、目錄、表格、callout。
   - 定義 AI MV report 要保留的互動區塊與要統一的視覺規則。

2. 設計背景 token
   - 定義白藍預設背景。
   - 定義格線、點陣、橫線、無紋路四種模式。
   - 定義 `--report-bg-*` 與 `--report-*` 變數。

3. 新增 `assets/report-theme.js`
   - 先只做讀取設定與套用 CSS variables。
   - 不先做首頁 UI，避免同時改太多。

4. 接入測試頁
   - 先接到首頁與 AI MV report。
   - 確認背景變成白藍淡格線，且列印模式仍維持白底。

#### 第 1 週交付物

- `docs/report-visual-standard.md`
- `assets/report-theme.js`
- 首頁與 AI MV report 先套用新的背景基礎。
- 截圖比對：桌機、手機、列印模式。

#### 第 1 週驗收條件

- AI MV report 背景比現在更白、更藍。
- 背景不是寫死在單一頁面，而是由共用設定控制。
- 不影響現有登入、目錄、右下角 action button、checklist。

### 第 2 週：所有 report 視覺一致化

目標：把共用背景套到所有 report，並讓 AI MV report 的色彩、表格、互動區塊更貼近其他三份 report。

#### 工作項目

1. 全站背景接入
   - `index.html`
   - `login.html`
   - `templates/report-template.html`
   - `reports/2026-05-12-nuva-new-taipei.html`
   - `reports/2026-05-22-itri-n8n-lv1.html`
   - `reports/2026-05-27-ai-experience-sharing.html`
   - `reports/2026-05-27-uni-president-ai-workshop-proposal.html`

2. AI MV report 視覺調整
   - 重整主色：以白色、深墨色與藍色為主。
   - 移除橘紅色，章節編號與警示重點統一改用藍色。
   - 表格改成跟其他 report 一樣節制，不刻意滿版。
   - prompt 區與 checklist 保持可操作，但邊框、間距、標題層級一致。
   - 影片區保留首頁可播放，但外框語言與 report 風格一致。

3. 共用元件視覺同步
   - `assets/report-reader.js` 的左側目錄背景與選取狀態吃共用變數。
   - `assets/report-actions.js` 的 action button 吃共用藍色與紙色。
   - 私密 report 登入擋板改成同一套背景。

4. 檢查既有功能
   - 首頁搜尋。
   - 登入後公開/不公開切換。
   - report 目錄。
   - AI MV checklist。
   - prompt 複製與編輯功能。
   - 列印文件。

#### 第 2 週交付物

- 所有頁面使用同一套背景設定。
- AI MV report 完成第一版視覺一致化。
- `templates/report-template.html` 更新，未來新 report 自動採用同一套背景。
- 完整 QA 截圖：首頁、登入頁、四份 report、手機版。

#### 第 2 週驗收條件

- 四份 report 放在一起看時，像同一個品牌系統下的不同主題。
- AI MV report 不失去教學與互動功能。
- 背景調淡後不影響文字閱讀。
- 列印或另存 PDF 不出現大面積格線底。

### 第 3 週：登入後背景控制台

目標：讓使用者登入首頁後，可以自己調整所有頁面的背景紋路、顏色、透明度與亮暗。

#### 工作項目

1. 首頁新增背景設定區
   - 未登入：只顯示「登入」。
   - 已登入：顯示背景設定控制台與目前公開狀態管理。
   - 控制台不要做成滿版大區塊，維持首頁現有報告感。

2. 控制項實作
   - 紋路：格線、點陣、橫線、無紋路。
   - 底色：色票與 color input。
   - 紋路透明度：slider。
   - 亮度：slider。
   - 格線大小：slider。
   - 重設預設值：button。

3. 即時預覽
   - 在首頁調整時立即套用。
   - 儲存到 `localStorage` 後，重新開 report 也會使用同一設定。
   - 若瀏覽器停用 `localStorage`，仍保留本頁即時效果。

4. 管理體驗整理
   - 背景設定與公開狀態管理分成清楚的兩塊。
   - 保留使用者原本要求：關掉頁面後需要重新登入。
   - 背景設定本身可保留在同一台瀏覽器，不跟登入狀態綁死。

5. 驗證與文件
   - 更新維運文件，說明背景設定如何重設。
   - 加入 release checklist。
   - 用瀏覽器測試不同背景設定下的四份 report。

#### 第 3 週交付物

- 首頁登入後背景控制台。
- 全站背景設定可跨頁保存。
- 預設白藍格線背景。
- 更新 `docs/maintenance-playbook.md` 或新增背景設定說明。
- 最終 QA 截圖與驗收紀錄。

#### 第 3 週驗收條件

- 登入後能調整背景紋路、顏色、透明度、亮度與格線大小。
- 調整後首頁、登入頁、所有 report 都套用同一背景。
- 重設後回到白藍淡格線。
- 未登入使用者看不到背景控制台。
- 關閉頁面後再次進入首頁仍需重新登入。
- 不破壞目前 private report、公開切換、搜尋、report 目錄與列印。

## 5. 建議實作順序

建議依照以下順序動手，避免一次改太多造成難以回溯：

1. 新增 `assets/report-theme.js`。
2. 在 `index.html` 與 AI MV report 接入並驗證。
3. 把背景設定擴展到 `login.html`、其他 report、template。
4. 調整 AI MV report 的色彩與局部元件。
5. 再做首頁登入後背景控制台。
6. 最後補文件與驗證腳本。

## 6. 風險與處理方式

| 風險 | 可能影響 | 處理方式 |
| --- | --- | --- |
| 每份 report CSS 都是 inline | 全站一致化容易漏改 | 先抽共用 script 與 CSS variables，再逐頁替換背景入口。 |
| 背景太明顯 | 影響 report 閱讀與列印 | 預設使用低 opacity，列印模式強制白底。 |
| 控制台設定壞掉 | 全站背景可能異常 | 設定讀取時做數值範圍限制，錯誤時回預設值。 |
| 舊 report 視覺差異大 | 一致化後可能失去原主題特色 | 只統一背景、紙張、線條與共用工具，不抹平每份 report 的主色。 |
| localStorage 被清掉 | 背景設定消失 | 使用預設白藍格線，不影響閱讀。 |
| 登入狀態與背景設定混淆 | 使用者以為背景也需要重新登入才保存 | 登入狀態維持 session，背景偏好維持 local；文件與 UI 清楚標示。 |

## 7. 不建議這次一起做的事

- 不建議把所有 report 改成 React 或其他框架，成本高且目前需求不需要。
- 不建議新增後端登入系統，現在需求是本機或靜態頁面的管理體驗。
- 不建議把每份 report 的內容結構一起大改，容易偏離「背景與風格一致化」主線。
- 不建議把背景控制做得像完整 theme builder，第一版先完成常用選項即可。

## 8. 最終完成樣貌

三週完成後，首頁、登入頁與所有 report 會共享同一套白色與藍色的清淡紙面背景系統。AI MV report 會保留現在好用的教學互動與格線氛圍，但整體字級、色彩、線條、表格與共用工具會更像同一組 report。登入首頁後，管理者可以直接調整背景的紋路、顏色、透明度、亮度與密度，並即時套用到所有頁面。
