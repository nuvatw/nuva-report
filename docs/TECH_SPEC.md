# Tech Spec

## Current Stack

- Static HTML files in `index.html`, `login.html`, and `reports/`.
- Shared scripts in `assets/`.
- Report index generation in `scripts/build-report-index.mjs`.
- The newest n8n report is generated by `scripts/build-n8n-deployment-course-report.mjs`.
- Data output is `data/reports.json` and `data/reports.js`.
- Validation scripts are plain Node scripts.

## Architecture Decision

Keep the current static stack. It already supports static export, local preview, Vercel deployment, metadata extraction, visual checks, and PDF-friendly pages. A Next.js migration would add build complexity without solving a current blocker.

Rollback note: if the visual upgrade creates unexpected static-script coupling, revert the latest CSS and JS changes while keeping the docs. The data/index pipeline can remain unchanged.

## Reusable Frontend Units

- Hub shell in `index.html`.
- Report reader, search, and progress in `assets/report-reader.js`.
- Report action bar in `assets/report-actions.js`.
- Background tokens in `assets/report-theme.js`.
- Generated report components in `scripts/build-n8n-deployment-course-report.mjs`, including tables, callouts, chapter shells, phase cards, week cards, diagrams, and source notes.

## Data Flow

1. Report HTML contains `report-meta`.
2. `npm run build` parses reports, injects shared report scripts, counts `.page` nodes, and writes data files.
3. `index.html` reads `data/reports.js` and renders searchable/filterable cards.
4. Report pages load reader and action scripts for TOC, search, visitor lock, print, and course link.

## Accessibility Requirements

- No console errors during screenshot checks.
- Keyboard access for filters, cards, reader search, and action buttons.
- Focus-visible styles for every button/link.
- Fixed controls should not trap focus.
- Mobile layouts must avoid horizontal overflow.

## Performance Requirements

- Keep the static no-bundler path.
- Avoid large framework dependencies.
- Use CSS transitions and IntersectionObserver instead of scroll event state loops.
- Keep generated report HTML readable even without JavaScript.
- Existing reports should remain reachable from the index.

## Print Requirements

- `@media print` must hide fixed reader/action UI.
- Report pages should render all important content.
- Screen-only controls must not consume paper space.
- Tables and cards should avoid clipping and preserve headers.
- The PDF workflow should use browser print or Playwright-assisted checks.

